# UI/UX Guidelines & Design System

This document serves as the single source of truth for the application's user interface, theming engine, and design standards. It is intended for developers and designers ensuring consistency across the application.

## 1. Theming Engine (Light & Dark Mode)

The application uses a robust theming engine built on **CSS Custom Properties (Variables)**.
- **Do not hardcode hex values** in components.
- Use semantic variables (e.g., `var(--bg-surface)`) rather than descriptive ones.
- Dark mode is handled via `[data-bs-theme='dark']` on the root element.
- **Persistence:** User preference is synchronized to `localStorage` to ensure the correct theme is applied immediately upon page load (preventing FOUC), even when using cloud-based storage.

## 2. Component Rules

### Buttons
- **Primary Actions:** Use `.btn-orange` (mapped to `--accent-primary`).
- **States:** All buttons must have visible distinct states:
  - **Hover:** Darker/intense variation.
  - **Active:** Pressed state visual feedback.
  - **Focus:** Clear focus ring (accessibility).
  - **Disabled:** Reduced opacity (0.65) and non-interactive cursor.

### Inputs
- **Focus Ring:** All inputs must have clear focus rings for accessibility. The application overrides default browser rings with a custom `--accent-primary` ring.
- **Labels:** Always use proper `<label>` or `aria-label`.

### Cards
- **Light Mode:** Use subtle shadows (`--shadow-card`) to create depth.
- **Dark Mode:** Avoid black shadows on dark backgrounds. Depth is communicated via lighter background overlays (`--bg-surface` vs `--bg-body`).

### Typography
- **Font Family:** `Inter`, sans-serif.
- **Scale:** Use fluid typography variables (`--font-sm`, `--font-base`, `--font-xl`) which use `clamp()` to scale across devices.
- **Contrast:** Ensure high contrast ratios (WCAG AA standard minimum).

## 3. Accessibility Checklist

- [ ] **Semantic HTML:** Use `<main>`, `<nav>`, `<article>`, `<header>`, `<footer>` appropriately to define document structure.
- [ ] **ARIA Labels:** Use `aria-label` or `aria-labelledby` where visual context (like icons) is insufficient.
- [ ] **Focus Management:** Ensure keyboard navigation is visible and logical. Focus rings must be preserved.
- [ ] **Contrast:** Verify text contrast against backgrounds in both Light and Dark modes.

## 4. Directory Structure

- **`src/index.css`**: The core definition of the design system. Contains all CSS variables, resets, and global overrides.
- **`src/components/`**: React components.
  - Components should generally avoid using separate CSS files unless necessary for complex, isolated layout logic.
- **`src/assets/`**: Static assets like images and global icons.
- **`src/domain/`**: Business logic and types (agnostic of UI).
- **`src/hooks/`**: Custom React hooks (e.g., `usePreferences`).
